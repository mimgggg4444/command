
## 🌟 깃허브 & 협업 필수 Git 명령어 순서

이 목록은 주로 **Fork/Clone**하여 다른 사람의 저장소(Original Repo, Upstream)와 자신의 저장소(Origin) 간에 동기화하며 작업하는 시나리오를 따릅니다.

### 1. 가져오기 (시작)

프로젝트를 시작하고 원격 저장소를 로컬로 가져오는 단계입니다.

| 명령어 | 설명 |
| :--- | :--- |
| `git clone [저장소 주소]` | 원격 저장소 전체를 로컬로 복사합니다. |
| `git remote add origin [내 깃허브 저장소 주소]` | (Fork의 경우) 로컬 저장소에 내 깃허브 주소를 `origin`으로 연결합니다. |
| `git remote add upstream [원본 저장소 주소]` | (Fork의 경우) **원본 저장소**를 `upstream`이라는 이름으로 로컬에 추가합니다. 이는 원본 저장소의 변경 사항을 추적하는 데 사용됩니다. |
| `git fetch upstream` | `upstream` 저장소에서 최신 변경 사항(커밋)을 로컬로 **다운로드만** 합니다 (병합은 하지 않음). |
| `git checkout main` | 또는 `master` 등 메인 브랜치로 이동합니다. |
| `git merge upstream/main` | 다운로드한 `upstream`의 변경 사항을 현재 브랜치(`main`)에 병합하여 로컬 `main` 브랜치를 최신 상태로 업데이트합니다. |
| `git push origin main` | 최신화된 로컬 `main` 브랜치를 내 원격 저장소(`origin`)에도 반영합니다. |

---

### 2. 변경 (작업) 및 반영

실제 코드를 수정하고 로컬 버전을 만든 후, 이를 원격 저장소에 올리는 단계입니다.

| 명령어 | 설명 |
| :--- | :--- |
| `git checkout -b [새 브랜치 이름]` | 작업 전에 **새로운 브랜치를 생성**하고 바로 이동합니다. (협업 시 필수) |
| `git status` | 수정한 파일 목록 확인 |
| `git add .` | 변경된 모든 파일을 스테이징 영역에 추가 |
| `git commit -m "[작업 내용 요약]"` | 스테이징된 변경 사항을 로컬에 커밋 (버전 생성) |
| `git push origin [새 브랜치 이름]` | 로컬에 만든 커밋을 내 원격 저장소(`origin`)의 해당 브랜치에 업로드 |
| **→ (GitHub 웹)** | 깃허브에서 **Pull Request (PR)**를 생성하여 원본 저장소에 병합 요청 |
| `git checkout main` | 작업이 완료되면 메인 브랜치로 돌아갑니다. |

---

### 3. 지우기 (삭제 및 취소)

실수로 커밋했거나, 원하지 않는 브랜치를 삭제할 때 사용합니다.

| 명령어 | 설명 |
| :--- | :--- |
| `git restore [파일명]` | 커밋하지 않은 로컬 변경 사항을 **취소**하고 파일의 마지막 커밋 상태로 되돌립니다. |
| `git reset HEAD^` | 마지막 커밋을 **취소**하고, 변경된 파일들은 로컬에 남겨둡니다 (Staging Area에서 제거됨). |
| `git reset [커밋 해시] --hard` | 특정 커밋으로 강제 **되돌리기** (이후의 모든 커밋과 로컬 변경 사항 **삭제** - **매우 위험!**) |
| `git revert [커밋 해시]` | 특정 커밋의 변경 내용을 되돌리는 **새로운 커밋을 생성**합니다 (기록 보존, 안전한 방법). |
| `git branch -d [브랜치 이름]` | 로컬 브랜치 삭제 (병합되지 않은 경우 `-D` 사용) |
| `git push origin --delete [브랜치 이름]` | 원격 저장소(`origin`)의 브랜치 삭제 |

---

### 4. 업데이트 (동기화)

작업 중이거나, 다른 사람이 변경한 내용을 로컬 작업 환경에 반영하는 단계입니다.

| 명령어 | 설명 |
| :--- | :--- |
| `git pull origin [브랜치]` | 내 원격 저장소(`origin`)의 최신 내용을 가져와 현재 브랜치에 **병합**합니다. |
| `git fetch upstream` | **원본 저장소(`upstream`)**의 변경 사항을 로컬에 다운로드합니다. |
| `git merge upstream/main` | 다운로드한 `upstream`의 내용을 로컬 브랜치에 **병합**합니다 (내 로컬 저장소를 최신화). |
| `git stash` | 현재 작업 중인 내용을 커밋하지 않고 **임시로 저장**하여 작업 공간을 깨끗하게 만듭니다. (업데이트나 브랜치 전환 전에 사용) |
| `git stash pop` | 저장해 두었던 임시 변경 사항을 다시 불러와 적용합니다. |

## 🌿 Git 브랜치(Branch) 관리 명령어 심화

### 1. 브랜치 생성 및 이동 (작업 환경 준비)

| 명령어 | 설명 |
| :--- | :--- |
| `git branch` | **로컬** 저장소의 브랜치 목록을 확인합니다. 현재 브랜치는 `*`로 표시됩니다. |
| `git branch [브랜치 이름]` | 현재 위치(커밋)에서 **새로운 브랜치만 생성**합니다. (이동은 하지 않음) |
| `git checkout [브랜치 이름]` | **기존의** 브랜치로 작업 환경을 **이동**합니다. |
| `git checkout -b [새 브랜치 이름]` | **새 브랜치를 생성**함과 동시에 해당 브랜치로 **즉시 이동**합니다. (가장 많이 사용) |
| `git switch [브랜치 이름]` | 브랜치 이동 (`checkout`의 대안으로 Git 최신 버전에서 권장됨). |
| `git switch -c [새 브랜치 이름]` | 새 브랜치를 만들고 이동 (`checkout -b`의 대안으로 Git 최신 버전에서 권장됨). |

### 2. 브랜치 작업 및 원격 반영 (내 작업 공유)

브랜치로 이동한 후 코드를 수정하고 이를 깃허브에 반영하는 순서입니다.

| 명령어 | 설명 |
| :--- | :--- |
| **(브랜치 이동 후)** | 코드 수정 및 파일 추가 작업 수행 |
| `git add .` | 변경된 파일을 스테이징 |
| `git commit -m "[작업 내용]"` | 로컬 커밋 생성 |
| `git push origin [내 브랜치 이름]` | 로컬에 생성한 커밋을 내 원격 저장소(`origin`)에 업로드하여 다른 사람과 공유합니다. |
| `git push -u origin [내 브랜치 이름]` | 해당 브랜치를 원격에 **처음** 올릴 때 사용합니다. (이후부터는 `git push`만 사용 가능) |

### 3. 다른 사람의 브랜치 가져오기 및 업데이트 (협업)

동료가 원격 저장소에 새로운 브랜치를 만들거나 기존 브랜치에 커밋을 푸시했을 때, 이를 내 로컬 환경으로 가져오는 방법입니다.

| 명령어 | 설명 |
| :--- | :--- |
| `git fetch` | **원격 저장소**(`origin` 및 `upstream` 등)의 모든 최신 변경 사항과 브랜치 정보를 **로컬로 다운로드**합니다. (작업 중인 파일에는 영향 없음) |
| `git branch -r` | 원격 저장소에 존재하는 브랜치 목록을 확인합니다 (예: `origin/feature-A`). |
| `git checkout [브랜치 이름]` | 이미 원격에서 가져와 로컬에 있는 브랜치로 이동합니다. |
| `git checkout -b [로컬 이름] origin/[원격 이름]` | 원격 저장소의 특정 브랜치(`origin/[원격 이름]`)를 로컬에 **새로운 브랜치**(`[로컬 이름]`)로 만들고 바로 이동합니다. |
| `git pull origin [브랜치 이름]` | 해당 브랜치에 대해 원격 저장소의 최신 커밋을 가져와 로컬 브랜치에 **병합(Merge)**합니다. (가장 흔한 업데이트 방법) |

### 4. 브랜치 삭제 및 정리

| 명령어 | 설명 |
| :--- | :--- |
| `git branch -d [브랜치 이름]` | 로컬 브랜치 삭제 (병합이 완료된 경우만 삭제됨) |
| `git branch -D [브랜치 이름]` | 로컬 브랜치 **강제 삭제** (병합 여부와 상관없이 삭제됨 - 주의 필요) |
| `git push origin --delete [브랜치 이름]` | **원격 저장소**(`origin`)의 브랜치를 삭제합니다. |
